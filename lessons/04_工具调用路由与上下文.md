# 工具调用、路由和上下文

这一章把工具调用、工具路由和上下文管理放在一起看。

前面几章已经讲了模型怎么回答、RAG 怎么检索。接下来要处理的是：问题来了以后，程序到底该查资料、调函数，还是先追问用户？如果聊天已经进行了很多轮，又该保留哪些历史？

## 主要内容

- 把普通 Python 函数写成工具。
- 给工具补上名称、说明和参数要求。
- 用规则先把问题分到大致工具组。
- 用关键词和简化向量分数找候选工具。
- 只把少量工具 schema 交给模型。
- 从用户问题里取出工具参数。
- 把多轮对话压成短记忆。
- 串起一个最小任务执行链。
