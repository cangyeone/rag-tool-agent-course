## 讲解主线

这一部分用于解释 AI、传统机器学习、深度学习、大模型之间的关系，并把字符、token、messages、流式输出等基础概念落到可运行代码上。

## 推荐运行顺序

```text
code/lesson_01_输入输出与对话格式/
├── 02_字符进入模型前长什么样.py    → 分词演示：字符→token→id，BPE 类比
├── 04_think_标签清理.py            → 清理模型输出中的 think/reasoning 标签
├── 05_流式输出体验.py              → 词级/句级流式输出，SSE 协议
└── 06_客服边界检查.py              → 五维度加权质量评分体系

code/lesson_02_本地模型与生成原理/
├── 01_tokenizer_观察.py             → 真实分词器与简化分词器对比
├── 02_最小_next_token_循环.py       → Next-token 预测循环，temperature 类比
├── 03_Qwen_本地模型加载检查.py      → 本地模型文件完整性校验
├── 04_Qwen_原始逐步输出.py          → 原始生成 vs chat 模板生成对比
├── 05_Qwen_chat模板与API.py         → Chat 模板渲染，本地调用 vs 云端 API
├── 06_Ollama_调用兜底.py            → Ollama 实际 API 调用与回退机制
├── 07_HTTP_服务接口草图.py          → Flask 服务，含 5 个路由
├── 08_模型后端切换.py               → BackendRouter 类，4 个后端自动回退
├── 09_GPR_与多模态安全样例.py       → 多模态融合在安全检查中的应用
└── 10_Qwen_多模态图像理解.py        → 本地 VL 模型实际处理图片（需 Qwen2.5-VL）
```

## 运行提示

涉及 Qwen 的脚本会自动向上查找：

```text
open_models/Qwen3.5-0.8B
```

如果本机算力有限，可以先运行 tokenizer、messages、流式输出等轻量脚本，再运行本地模型加载脚本。
