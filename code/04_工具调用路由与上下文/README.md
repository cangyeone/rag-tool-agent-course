# 04 工具调用、路由和上下文

这一章把三件事放在一起讲：

- 工具：把“查资料、算结果、生成摘要”这些能力写成函数。
- 路由：问题来了以后，先判断该找哪类工具。
- 上下文：多轮对话时，只把有用的信息留下来。

这几件事通常会一起出现。工具太多时，不能一次性全部交给模型；对话太长时，也不能把所有历史都塞回去。代码里的例子都用普通 Python 写，不依赖平台。

## 文件顺序

```text
01_把函数当工具.py
02_工具清单和参数检查.py
03_先用规则缩小范围.py
04_关键词和向量一起找工具.py
05_只把少量schema给模型.py
06_从问题里取参数.py
07_短记忆和摘要.py
08_一个小型任务执行器.py
```

## 先看这几个概念

- 工具：一个可以被程序调用的函数。
- 工具说明：告诉模型或路由器，这个工具能做什么。
- 参数：调用工具时需要传入的信息。
- 路由：先判断问题类型，再选工具。
- schema：工具参数的格式说明。
- 记忆：从历史对话里留下来的关键信息。
