## 本节课目标

本节课按 5 个 demo 展开。每个 demo 都能单独运行，适合先讲 3-5 分钟，再可改一个参数观察输出。

## 演示脚本

### Demo 1: 工具 schema 设计

脚本：`01_工具_schema_设计.py`

演示重点：给每个工具定义 name、description、args、return 字段。

运行方式：

```bash
python 01_工具_schema_设计.py
```

### Demo 2: 注册表校验

脚本：`02_注册表校验.py`

演示重点：检查工具是否有说明、函数和必需参数。

运行方式：

```bash
python 02_注册表校验.py
```

### Demo 3: 工具执行器

脚本：`03_工具执行器.py`

演示重点：按 tool name 和 args 统一执行工具。

运行方式：

```bash
python 03_工具执行器.py
```

### Demo 4: 工具错误处理

脚本：`04_工具错误处理.py`

演示重点：模拟缺参数、工具不存在、结果为空三类错误。

运行方式：

```bash
python 04_工具错误处理.py
```

### Demo 5: 工具测试集

脚本：`05_工具测试集.py`

演示重点：用多条问题检查工具是否按预期返回。

运行方式：

```bash
python 05_工具测试集.py
```

## 推荐节奏

- Demo 1：建立本节课概念入口。
- Demo 2-3：展开核心代码，可改输入或参数。
- Demo 4：把代码能力接到真实应用视角，例如服务、检索、工具或工作流。
- 最后一个 Demo：做小复盘，检查输出是否符合业务边界。

## 统一修改点

- 改用户问题，观察检索、路由或质检输出是否变化。
- 改规则资料，观察命中片段和回答依据是否变化。
- 改工具参数，观察结构化结果和最终回答是否变化。
- 把终端输出发给课堂助教，让它解释这个 demo 在知识地图中的位置。
