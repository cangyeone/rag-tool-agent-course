## 讲解主线

这一部分强调 Router 的工程价值：不是把所有工具一次性给模型，而是先判断任务方向，再把工具候选从几百个压缩到少量可用工具。lesson_01 讲基础分流，lesson_02 聚焦“几百个工具如何调用”：工具分组、规则过滤、向量召回、LLM rerank、schema 注入和调用审计。

## 推荐运行顺序

```text
code/lesson_01_Router基础/
├── 01_为什么需要Router.py           → 多工具场景下的分流需求

├── 03_模型选择路由.py               → 根据任务复杂度选择不同模型

└── 05_路由与工具编排.py             → Router + Tool 的综合编排演示

code/lesson_02_模型路由与参数抽取/
├── 01_工具分组与轻量注册表.py           → 工具拆成轻量 tool card 和完整 full_schema
├── 02_规则过滤_先定位工具组.py         → 邮件/演示材料/代码仓库/知识库等强规则分组

├── 04_LLM重排_判断工具是否真相关.py    → 判断召回工具是否真的适合当前任务
├── 05_schema注入_只给少量完整参数.py   → 只给最终 3-10 个工具完整 schema
```

## 运行提示

本组脚本以模拟业务工具为主，不连接真实业务系统。第 08 个脚本会使用本地 BGE-m3 和 FAISS 建真实向量库。课堂上可以修改用户问题，观察规则命中、BM25、向量召回、RRF 融合、rerank 排名和最终注入 schema 如何变化。
