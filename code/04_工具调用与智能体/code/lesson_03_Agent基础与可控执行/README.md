## 本节课目标

本节课按 6 个 demo 展开。每个 demo 都能单独运行，适合先讲 3-5 分钟，再可改一个参数观察输出。

## 演示脚本

### Demo 1: ReAct 轨迹

脚本：`01_ReAct_轨迹.py`

演示重点：展示 Thought、Action、Observation、Answer 的基本结构。

运行方式：

```bash
python 01_ReAct_轨迹.py
```

### Demo 2: 计划器 Planner

脚本：`02_计划器_Planner.py`

演示重点：先列出需要哪些步骤，再逐步执行。

运行方式：

```bash
python 02_计划器_Planner.py
```

### Demo 3: 多轮状态记忆

脚本：`03_多轮状态记忆.py`

演示重点：保存用户意图、槽位、工具结果和最终回答。

运行方式：

```bash
python 03_多轮状态记忆.py
```

### Demo 4: 安全边界 Guardrail

脚本：`04_安全边界_Guardrail.py`

演示重点：拦截抢票、绕过系统、承诺成功等不合规表达。

运行方式：

```bash
python 04_安全边界_Guardrail.py
```

### Demo 5: Agent 小型评测

脚本：`05_Agent_小型评测.py`

演示重点：用多场景问题检查 Agent 是否稳。

运行方式：

```bash
python 05_Agent_小型评测.py
```

### Demo 6: Agent harness 与未来软件形态

脚本：`06_Agent_harness_与未来软件形态.py`

演示重点：把工具权限、轨迹日志、人工确认和工作流入口整理成 Agent 外层框架。

运行方式：

```bash
python 06_Agent_harness_与未来软件形态.py
```

## 推荐节奏

- Demo 1：建立本节课概念入口。
- Demo 2-3：展开核心代码，可改输入或参数。
- Demo 4：把代码能力接到真实应用视角，例如服务、检索、工具或工作流。
- 最后一个 Demo：做小复盘，检查输出是否符合业务边界。

## 统一修改点

- 改用户问题，观察检索、路由或质检输出是否变化。
- 改规则资料，观察命中片段和回答依据是否变化。
- 改工具参数，观察结构化结果和最终回答是否变化。
- 把终端输出发给课堂助教，让它解释这个 demo 在知识地图中的位置。
