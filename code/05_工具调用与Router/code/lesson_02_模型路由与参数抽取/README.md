# 几百个工具如何调用

这一节不再重复第 04 章的“单个工具调用流程”。第 04 章已经讲清楚：模型如何选择工具、程序如何执行工具、工具结果如何回传模型。

本节关注更真实的工程问题：

```text
系统里有几十个、几百个工具时，模型到底该看到哪些工具？
```

如果把全部工具的完整 schema 一次性塞给模型，会带来几个问题：

- 输入太长，成本高，延迟大。
- 无关工具太多，模型更容易误选。
- 工具参数定义太多，arguments 更容易生成错。
- 出问题后难以判断是召回、重排、schema 还是模型选择错了。

所以这一节改成“几百个工具如何调用”的完整链路。

## 本节主线

```text
用户问题
  ↓
1. 工具分组与轻量注册表
  ↓
2. 规则过滤：邮件 / 演示材料 / 代码仓库 / 知识库
  ↓
3. 向量召回：从 tool descriptions 中找 top-k
  ↓
4. LLM rerank：判断工具是否真的相关
  ↓
5. schema 注入：只给最终 3-10 个工具的完整参数定义
  ↓
6. 工具调用、执行与审计
  ↓
7. 端到端调用链路
  ↓
8. 真实向量库 + 关键词混合召回
```

## 演示脚本

### Demo 1：工具分组与轻量注册表

脚本：`01_工具分组与轻量注册表.py`

讲解重点：

- 几百个工具要先分组。
- 每个工具维护两份信息：轻量 tool card 和完整 full_schema。
- 前几步只用轻量描述，最后才注入完整 schema。

运行：

```bash
python 01_工具分组与轻量注册表.py
```

### Demo 2：规则过滤

脚本：`02_规则过滤_先定位工具组.py`

讲解重点：

- 用户说“邮件” → Gmail tool group
- 用户说“演示材料” → presentation tool group
- 用户说“代码仓库” → code tools
- 用户说“知识库” → RAG tools

运行：

```bash
python 02_规则过滤_先定位工具组.py
```

### Demo 3：向量召回

脚本：`03_向量召回_从描述中找topk.py`

讲解重点：

- 规则过滤后仍然可能有几十个候选工具。
- 向量召回只看工具的轻量 description 和 short_schema。
- 真实系统可替换成 BGE-m3、FAISS、DashVector、OpenSearch。

运行：

```bash
python 03_向量召回_从描述中找topk.py
```

### Demo 4：LLM rerank

脚本：`04_LLM重排_判断工具是否真相关.py`

讲解重点：

- 向量召回只是“像不像”。
- LLM rerank 判断“该不该用”。
- 比如用户说“找邮件里的培训课程材料”，当前动作是查邮件，不是生成 演示材料。

运行：

```bash
python 04_LLM重排_判断工具是否真相关.py
```

### Demo 5：schema 注入

脚本：`05_schema注入_只给少量完整参数.py`

讲解重点：

- 只给最终 3-10 个工具的完整参数定义。
- 不要把几百个 full_schema 全塞给模型。
- 对比全量 schema 和最终注入 schema 的长度。

运行：

```bash
python 05_schema注入_只给少量完整参数.py
```

### Demo 6：调用执行与审计日志

脚本：`06_调用执行与审计日志.py`

讲解重点：

- 几百个工具场景必须记录选择过程。
- 审计日志包括规则命中、向量召回、rerank 理由、注入 schema、模型 tool call、工具结果。
- 工具误调用时，靠日志定位问题。

运行：

```bash
python 06_调用执行与审计日志.py
```

### Demo 7：端到端调用链路

脚本：`07_几百个工具_端到端调用链路.py`

讲解重点：

- 把 1-6 步全部串起来。
- 展示“几百个工具 → 少量候选工具 → 一个工具调用”的压缩过程。
- 这才是本节和第 04 章的区别。

运行：

```bash
python 07_几百个工具_端到端调用链路.py
```

### Demo 8：真实向量库与关键词混合召回

脚本：`08_真实向量库与关键词混合召回.py`

讲解重点：

- 用本地 `open_models/bge-m3` 生成工具描述向量。
- 用 FAISS 建一个真实工具向量库。
- 用 BM25 做关键词检索。
- 用 RRF 融合关键词结果和向量结果。
- 最后仍然只注入少量 full_schema。

运行：

```bash
python 08_真实向量库与关键词混合召回.py
```

## 本节产出

学完后应能说清：

- 为什么不能把几百个工具全部给模型。
- 什么是轻量 tool card。
- 规则过滤、向量召回、LLM rerank 分别解决什么问题。
- 为什么完整 schema 要最后注入。
- 工具调用日志应该记录哪些信息。
