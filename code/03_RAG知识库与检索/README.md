## 讲解主线

这一部分对应 RAG 的全流程：资料进入系统 → 切片（overlap）→ 关键词检索（TF→TF-IDF→BM25）→ 向量化 → 向量库 → 混合检索 → RRF 排名融合 → 工具函数 → 回答质检。

## 推荐运行顺序

```text
code/lesson_01_资料整理切片关键词检索/
├── 01_资料清单盘点.py              → 文件系统扫描，构建文档清单
├── 02_文档切片.py                  → 3 种切片策略对比演示
├── 03_overlap_作用.py              → overlap 参数并排对比，防止语义断裂
├── 04_关键词检索.py                → TF→TF-IDF→BM25 递进演示
└── 05_检索结果解释.py              → 置信度打分与结果分级

code/lesson_02_向量检索与向量库/
├── 01_文本向量化.py                → 基于 hash 的向量模拟，理解 embedding 原理
├── 02_余弦相似度.py                → 余弦相似度数学推导与可视化
├── 03_向量库写入.py                → 索引构建，多种格式对比
├── 04_向量库读取检索.py            → 磁盘加载、缓存、ANN 简介
└── 05_召回质量检查.py              → Precision@K、Recall@K、MRR、NDCG

code/lesson_03_混合检索与RRF/
├── 01_关键词加向量混合检索.py      → 线性融合，权重调优演示

├── 03_工具函数最小版.py            → 工具 schema 设计，RAG 与 tool 对比
```

## 运行提示

本组脚本大部分使用可解释的模拟向量，便于课堂讲清楚检索逻辑。后续可以把模拟向量替换为 `open_models/bge-m3` 生成的真实 embedding。按 lesson_01 → lesson_02 → lesson_03 顺序运行，每个 lesson 内按文件序号递增运行。
