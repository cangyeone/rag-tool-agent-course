# OpenCode 学习指南

这套课程推荐学员使用 OpenCode 来阅读、修改和运行脚本。OpenCode 的作用不是替代学习，而是把“看不懂代码”和“运行报错”这两件事变得更容易处理。

## 基本用法

1. 用 OpenCode 打开仓库根目录。
2. 先读 `README.md` 和 `COURSE_OUTLINE.md`。
3. 进入 `code/`，按编号选择脚本。
4. 打开同名 `.md` 文档，看清楚学习目标、运行方式和可修改点。
5. 运行脚本，观察输出。
6. 修改一个参数，再运行一次。

## 推荐提问方式

```text
请按“输入、处理、输出”解释当前脚本。
请指出这个脚本里最适合我修改的 3 个参数。
请把示例数据换成一个通用客服 FAQ 场景。
请帮我把这个脚本改成读取 data/demo_docs 里的 Markdown 文件。
请根据终端报错修复代码，但不要写入任何 API Key。
```

## 安全要求

- 不要把 API Key 粘贴到代码里。
- 不要提交 `.env` 文件。
- 不要上传企业内部资料、真实用户数据或未授权文档。
- 让 OpenCode 修改代码后，要自己阅读差异，再运行验证。

## 课堂练习建议

每个脚本至少做三次操作：

1. 原样运行，确认脚本能跑通。
2. 修改输入问题或示例文档。
3. 修改一个参数，例如 `top_k`、`chunk_size`、`temperature` 或工具描述。

这样能把“看过”变成“会改”。
