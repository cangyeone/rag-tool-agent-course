# 代码 Markdown 说明索引

本仓库的代码目录采用一组固定规则：

- Python 脚本使用 `.py` 后缀。
- 脚本说明使用同名 `.md` 后缀。
- 例如：`01_资料清单盘点.py` 对应 `01_资料清单盘点.md`。

## 使用方式

1. 先读章节 README，了解这一组脚本要解决的问题。
2. 再读脚本旁边的同名 Markdown，确认目标、输入、输出和可修改点。
3. 打开 Python 脚本，对照 Markdown 阅读代码。
4. 使用 OpenCode 修改脚本：可以改问题、改参数、改检索资料、改工具描述。
5. 运行脚本，观察输出变化。

## 给 OpenCode 的常用指令

```text
请阅读这个 Markdown 任务书，然后检查同名 Python 脚本是否完整实现了要求。
代码要适合初学者阅读：少用复杂封装，按步骤顺序写，保留必要注释。
请不要写入任何 API Key。
```

```text
请根据这个 Markdown 任务书，为同名 Python 脚本增加一个可修改参数。
同时在 Markdown 中补充这个参数的作用、默认值和修改建议。
```

## 命名检查

公开课版本统一使用脚本同名 Markdown，例如 `01_资料清单盘点.md`。
