```

```

#### 步骤 4：查看回答

右侧中间区域显示对话历史：

```
┌─────────────────────────────────────────┐
│  👤 您：                                 │
│  RAG 的切片和 overlap 有什么作用？      │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  🤖 AI：                                 │
│  RAG（Retrieval-Augmented Generation）  │
│  中的切片（chunking）和重叠（overlap）   │
│  是两个重要的预处理步骤：                │
│                                         │
│  1. **切片的作用**：                     │
│     - 将长文档分割成小片段              │
│     - 便于向量检索和上下文匹配          │
│     - 提高检索精度和效率                │
│                                         │
│  2. **Overlap 的作用**：                 │
│     - 保持相邻切片之间的连续性          │
│     - 避免关键信息被切分丢失            │
│     - 提升语义完整性                    │
│                                         │
│  推荐设置：                              │
│  - 切片长度：2000 字符                  │
│  - Overlap：500 字符（25%）             │
└─────────────────────────────────────────┘
```

同时，左侧对话标题自动更新：

## 🎯 核心特性总结

### ✅ 多会话管理

- **无限对话**：可以创建任意数量的对话
- **自动保存**：每次操作自动保存到磁盘
- **即时切换**：点击即可切换，无延迟
- **持久化存储**：刷新页面、重启服务都不丢失

### ✅ 智能标题生成

- **自动生成**：取第一条用户消息的前 20 字
- **实时更新**：发送第一条消息后立即更新
- **易于识别**：快速找到想要的对话

### ✅ 友好的视觉反馈

- **颜色编码**：蓝色=用户，绿色=AI
- **加载动画**：检索和生成时显示 spinner
- **高亮当前**：选中的对话蓝色背景
- **气泡样式**：清晰的对话边界

### ✅ 灵活的检索配置

- **实时调整**：滑块拖动立即生效
- **悬停提示**：鼠标悬停显示帮助文本
- **个性化**：每个用户可以有自己的偏好设置

### ✅ 数据安全

- **本地存储**：所有数据保存在本地
- **JSON 格式**：易于备份和迁移
- **Git 忽略**：`.gitignore` 自动排除敏感数据

## 📊 性能表现

| 操作 | 耗时 | 说明 |
|------|------|------|
| 启动服务 | 2-5 秒 | 取决于索引大小 |
| 加载索引 | 1-3 秒 | 首次加载，后续缓存 |
| 检索片段 | 0.5-1 秒 | 取决于 top_k 和向量维度 |
| 生成回答 | 3-8 秒 | 取决于回答长度和网络速度 |
| 切换对话 | < 0.1 秒 | 几乎无延迟 |
| 新建对话 | < 0.1 秒 | 几乎无延迟 |

## 🔍 调试技巧

### 查看会话文件

```bash
ls -lh sessions/
# session_1234567890.json
# session_0987654321.json
```

### 查看会话内容

```bash
cat sessions/session_1234567890.json | python -m json.tool
```

### 手动备份

```bash
tar -czf sessions_backup_$(date +%Y%m%d).tar.gz sessions/
```

### 清空所有对话

```bash
rm -rf sessions/*
# 刷新页面，会自动创建新对话
```
