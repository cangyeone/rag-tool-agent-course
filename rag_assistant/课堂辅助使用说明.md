## 适合可提问什么

这个机器人适合作为课堂辅助工具，不替代授课讲解，主要负责帮参与者快速定位课程资料、解释实操步骤、排查常见报错。

推荐问题：

- 今天课程的实操主线是什么？
- RAG 问答机器人从提问到回答经过哪些步骤？
- 可视化工具 里知识库节点和 DeepSeek 节点分别负责什么？
- 运行某个 Python 脚本前需要准备什么？
- 这个报错可能是什么原因？我应该先检查哪里？
- 某段代码可以改哪些参数做实验？
- 检索结果不准时，应该检查切片、关键词还是输入上下文？

## 参与者提问格式

为了让回答更准确，建议参与者尽量带上上下文：

```text
我在运行 01_6月22日_RAG工具使用培训/code/10_示例业务系统知识库加工具问答.py，
问题是：检索结果不符合预期。
我已经修改了 query=......
应该先检查什么？
```

如果是报错，建议直接贴：

```text
我运行到第几步：
报错信息：
我当前使用的系统：
我已经尝试过：
```

## 课堂使用建议

上课时可以把它放在三个位置使用：

1. 开课前：可提问环境准备、安装步骤、当天课程主线。
2. 实操中：可带着文件名、代码片段、报错信息提问。
3. 课后：可复盘 RAG、可视化工具、DeepSeek、工具调用之间的关系。

建议提醒参与者：不要把真实 API Key、账号密码、内部系统地址贴到机器人里。遇到密钥配置问题，只描述“环境变量是否设置”“可视化工具 凭据是否可用”即可。

## 重新导入知识库

如果课程资料有更新，先确认 可视化工具 已启动，然后在项目根目录运行：

```bash
python rag-tool-agent-course/RAG_问答机器人/create_low_code_platform_rag_bot.py
```

这个脚本会重新扫描 `rag-tool-agent-course` 目录，把课程文档和课堂代码整理进 可视化工具 知识库，并更新工作流草稿。更新后建议在 可视化工具 页面点击“发布”，让聊天入口使用最新版本。
