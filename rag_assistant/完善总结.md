## 🔧 核心功能

### 会话管理系统

```python
# 会话数据结构
{
    "id": "session_1234567890",           # 唯一 ID
    "title": "RAG 的切片有什么作用？",     # 自动生成标题
    "created_at": "2026-06-23T15:18:00",   # 创建时间
    "messages": [                          # 消息列表
        {
            "role": "user",                # 角色：user/assistant
            "content": "RAG 的切片...",     # 消息内容
            "timestamp": "2026-06-23..."    # 时间戳
        },
        ...
    ]
}
```

**关键函数**：
- `create_new_session()` - 创建新会话
- `load_session(session_id)` - 加载会话数据
- `save_session(session_id, data)` - 保存会话数据
- `list_sessions()` - 列出所有会话（按时间倒序）
- `delete_session(session_id)` - 删除会话
- `add_message_to_session(session_id, role, content)` - 添加消息

### 数据存储

- **索引数据**：`storage/` 目录（chunks.json + embeddings.npy）
- **会话数据**：`sessions/` 目录（自动生成，每个会话一个 JSON 文件）
- **持久化**：每次操作自动保存到磁盘，刷新页面不丢失

## 📊 技术细节

### 依赖包

```bash
streamlit          # Web 框架
numpy              # 数值计算
faiss-cpu          # 向量数据库
sentence-transformers  # 嵌入模型
requests           # HTTP 请求
rich               # 终端美化
jieba              # 中文分词
```

### 环境要求

- Python 3.8+
- DeepSeek API Key（在 `.env` 文件中配置）
- 已构建的 FAISS 索引（`storage/` 目录）

### 性能优化

- 使用 `st.cache_resource` 缓存 RAG 索引加载
- 会话数据采用 JSON 格式，读写快速
- 懒加载策略，按需加载会话数据

##  设计理念

### 1. 清晰的视觉层次
- 左侧深色背景区分功能区
- 右侧白色背景突出主要内容
- 使用颜色编码（蓝色=用户，绿色=AI）

### 2. 高效的操作流程
- 常用操作使用主要按钮（蓝色）
- 危险操作使用图标按钮（红色）
- 减少点击次数，提升效率

### 3. 友好的反馈机制
- 加载状态显示 spinner
- 错误消息使用红色警告框
- 成功操作即时刷新

### 4. 数据安全性
- 自动保存，防止数据丢失
- 本地存储，保护隐私
- 可手动备份 `sessions/` 目录
